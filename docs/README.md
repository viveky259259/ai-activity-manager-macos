## Conventions

- **PRDs** in `prds/` are intent-of-record per package. They're the canonical "what should this do" docs and are updated when scope changes.
- **`features.md`** is the rolling inventory of what actually ships in the current release.
- **`launch/`** is freezable per-release marketing copy, not living docs.
- New top-level docs go here, not in `launch/` or `prds/`.
